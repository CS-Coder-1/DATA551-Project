# Team Contract

We agree to the following to work well together:

1. **Work Distribution**  
   - Tasks assigned based on skills and current workload  
   - Use GitHub to track everything  
   - Rebalance if someone is overloaded

2. **Availability & Response Time**  
   - Reply to messages within a reasonable timeframe  
   - Announce absences / heavy weeks in advance

3. **Communication**  
   - Quick questions → group chat  
   - Code discussion → GitHub PR comments  

4. **Quality & Accountability**  
   - Submit clean, tested code  
   - Be honest about progress and problems  
   - Raise issues early and respectfully

5. **Conflict**  
   - Talk directly and constructively  
   - If needed, bring to group or ask instructor for help
